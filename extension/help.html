FetchGate — Host Setup

What is the native host?

FetchGate requires a native host — a small program that Firefox launches on your machine when you arm a tab. The extension alone can arm a tab and route requests through it; the host is the bridge between the extension and your own code. Without it, the tab will arm but has no way to receive requests from outside the browser.

Firefox communicates with the host over stdin/stdout (Firefox Native Messaging). Your code communicates with the host over TCP — or, with the embedded Python host, your script is the host and Firefox launches it directly.

Only one host can be active at a time. All three install to the same file: ~/.mozilla/native-messaging-hosts/fetchgate.json. To switch hosts, overwrite that file with the other template and reload the extension in about:debugging.

Java host

A persistent TCP server on localhost:9919. Callers connect from any language. Can run directly with a JDK or via Docker (no JDK needed).

Option A — Direct (JDK 21+ required)

1
Clone and compile 
git clone https://github.com/simddev/FetchGate
cd FetchGate
javac -d out src/*.java
2
Create a launcher script

Firefox must launch an executable — create ~/bin/fetchgate.sh:

#!/bin/bash
exec java -cp "/absolute/path/to/FetchGate/out" Main
chmod +x ~/bin/fetchgate.sh
3
Register the native host 
mkdir -p ~/.mozilla/native-messaging-hosts
cp fetchgate.json ~/.mozilla/native-messaging-hosts/fetchgate.json

Open ~/.mozilla/native-messaging-hosts/fetchgate.json and set the path:

"path": "/home/you/bin/fetchgate.sh"

Option B — Docker (no JDK required)

1
Clone and build the image 
git clone https://github.com/simddev/FetchGate
cd FetchGate
docker build -t fetchgate .
2
Create a launcher script

Create ~/bin/fetchgate.sh:

#!/bin/bash
exec docker run --rm -i --network=host fetchgate
chmod +x ~/bin/fetchgate.sh
3
Register the native host — same as Option A, step 3 above.

Smoke test

Arm a tab, then:

echo '{"method":"GET","url":"/robots.txt"}' | timeout 3 nc localhost 9919

You should get back one JSON line with status, headers, and body. timeout 3 is needed because the Java host keeps connections open — without it, nc hangs waiting for more data after the response.

Python TCP host

A pure-Python drop-in replacement for the Java host. Same TCP interface on localhost:9919 — any caller that works with the Java host works unchanged. No JDK needed.

1
Get the script 
git clone https://github.com/simddev/FetchGate
chmod +x FetchGate/host_py/fetchgate_tcp_host.py

Or copy host_py/fetchgate_tcp_host.py to a permanent location and chmod +x it there.

2
Register the native host 
mkdir -p ~/.mozilla/native-messaging-hosts
cp fetchgate_tcp_py.json ~/.mozilla/native-messaging-hosts/fetchgate.json

Set path to the absolute path of fetchgate_tcp_host.py:

"path": "/absolute/path/to/fetchgate_tcp_host.py"

Smoke test

Arm a tab, then:

echo '{"method":"GET","url":"/robots.txt"}' | nc localhost 9919

Python embedded host

Your Python script IS the native host. Firefox launches it when you arm a tab. No persistent server — the script runs once and exits. Ideal for targeted one-off data extraction jobs.

1
Copy the example script 
cp FetchGate/host_py/example.py ~/bin/my_scraper.py
chmod +x ~/bin/my_scraper.py
2
Edit your script

Keep the imports and FetchGate() constructor. Everything after is your code:

#!/usr/bin/env python3
import sys, os
sys.path.insert(0, "/absolute/path/to/FetchGate/host_py")
from fetchgate import FetchGate, FetchGateError

fg = FetchGate()

# Fetch mode — authenticated HTTP request via the tab
resp = fg.fetch({"method": "GET", "url": "/api/data"})
print(resp["status"], resp["body"])

# JS mode — run arbitrary JavaScript in the tab
resp = fg.fetch({"js": "return document.title"})
print(resp["result"])
stdout is redirected. After FetchGate() is constructed, sys.stdout is redirected to sys.stderr to prevent accidental print() calls from corrupting the NM stream. Use sys.__stdout__ to write output you want to capture from outside the browser.
3
Register the native host 
mkdir -p ~/.mozilla/native-messaging-hosts
cp fetchgate_py.json ~/.mozilla/native-messaging-hosts/fetchgate.json

Set path to your script:

"path": "/home/you/bin/my_scraper.py"

The script must be executable (chmod +x) and must have a shebang line (#!/usr/bin/env python3) — Firefox uses the shebang to launch it.

4
Run it

Navigate to the target website, then arm the tab (click the toolbar button). Firefox launches your script immediately. The badge turns ERR when your script finishes — that is expected (the NM connection closed because the script exited). Click the toolbar button again to re-arm and re-run.

Writing your own host

Any process that speaks Firefox Native Messaging can work with this extension. The wire format:

  • 4-byte little-endian unsigned length header + UTF-8 JSON payload
  • Firefox enforces a hard 1 MB per-message cap

The extension sends a request envelope and expects a matching reply:

// Request from the extension to your host
{"__fg_id": 1, "req": "{\"method\":\"GET\",\"url\":\"/api/data\"}"}

// Fetch mode reply — echo __fg_id, spread response fields
{"__fg_id": 1, "status": 200, "statusText": "OK", "headers": {...}, "body": "..."}

// JS mode reply
{"__fg_id": 1, "result": "..."}

// Error
{"__fg_id": 1, "error": "description"}

req is the caller's JSON serialized as a string so the extension can delegate JSON validation to JSON.parse(). Echo __fg_id in every reply so the extension can match each response to its originating request.

Troubleshooting

Badge shows ERR immediately after arming

The native host failed to start. Check:

  • ~/.mozilla/native-messaging-hosts/fetchgate.json exists
  • The path in that file is an absolute path to an executable file
  • The executable has chmod +x and (for Python scripts) a #!/usr/bin/env python3 shebang
  • You reloaded the extension after editing the manifest (go to about:debugging → This Firefox → FetchGate → Reload)

Open the background console for detailed errors: about:debugging → This Firefox → FetchGate → Inspect → Console.

Switching between hosts

# Switch to Java host
cp fetchgate.json ~/.mozilla/native-messaging-hosts/fetchgate.json
# edit "path" to point to fetchgate.sh

# Switch to Python TCP host
cp fetchgate_tcp_py.json ~/.mozilla/native-messaging-hosts/fetchgate.json
# edit "path" to point to fetchgate_tcp_host.py

# Switch to Python embedded host
cp fetchgate_py.json ~/.mozilla/native-messaging-hosts/fetchgate.json
# edit "path" to point to your script

After switching, reload the extension in about:debugging.
